# 機能一覧

## ダッシュボード

日々の生活に役立つ情報を表示するダッシュボード機能を提供します。ダッシュボードは
シックでシンプルなデザインを採用し、必要な情報を見やすく配置します。
表示言語は英語とします。
ダッシュボードの表示サイズは SXGA（1280x1024）を前提とします。

![デザイン（スクリーンショット）](design.png)

`design.png` は [designExample.html](designExample.html) のスクリーンショットです。
レイアウトや表示要素の正本は [designExample.html](designExample.html) とし、本ドキュメントはその内容を実装向けに文章化したものです。

## 表示構成

ダッシュボードは以下の 3 要素で構成します。

- 左: 写真カード
- 右上: 日時カード
- 右下: 天気カード

現状のコード実装と差分がある場合でも、本ドキュメントの内容を今後の表示仕様とします。

## レイアウト仕様

画面全体は 1280x1024 の固定サイズを前提にし、外周に 50px の余白を確保します。
主要レイアウトは 2 カラム構成とし、左カラムを 800px、右カラムを 400px 相当とします。
カラム間隔は 24px とします。

- 左カラム: 写真カード 1 枚を全面表示する主領域
- 右カラム: 上下 2 枚のカードを縦積みする情報領域
- 右カラム内のカード間隔: 24px
- 各カードの内側余白: 40px を基準とする
- 各カードの角丸: 24px 相当とする

実装では framebuffer 上の描画都合で数 px の調整は許容しますが、左右の領域比率、カードの上下構成、余白感は維持します。

## 表示内容

ダッシュボードは以下の情報を表示します。

- 日付
- 時刻
- 天気情報
- ローカルディレクトリに同期された画像

各要素の必須表示項目は後述します。

## 日時カード

日時カードは右上に配置します。表示内容は以下を必須とします。

- 上段に日付を 1 行で表示する
- 中央に時刻を大きく表示する

日付表示の仕様:

- 形式は `YYYY/MM/DD`
- タイムゾーンは設定されたローカルタイムゾーンを使用する
- 時刻より小さい文字サイズで表示する

時刻表示の仕様:

- 形式は `HH:MM:SS`
- 1 秒ごとに更新する
- 画面内で最も強い視認性を持つ表示要素として扱う

## 天気カード

天気カードは右下に配置します。表示内容は以下を必須とします。

- 天気アイコン
- 現在の気温（摂氏）
- 状態文言
- ロケーション文字列

表示仕様:

- アイコンはカード上部に配置する
- 気温はカード内で大きく表示する
- 状態文言は英語の短い語句を想定し、補助情報として表示する
- ロケーション文字列は最下段に表示する

データ取得の仕様:

- 天気情報は一定間隔で更新する（デフォルト 10 分）
- Open-Meteo API を使用する
- 取得に失敗した場合でも、最後に取得できたキャッシュがあればそれを表示する

## 写真カード

写真カードは左側の主領域に配置します。表示内容と振る舞いは以下の通りです。

- ローカルディレクトリに同期された画像を 1 枚表示する
- 複数枚ある場合はランダムに切り替える
- 切り替え時は主表示を阻害しない範囲で滑らかに見える遷移を想定する
- カード下端には進行状況を示す細いインジケータを置ける

画像ソースの前提:

- アプリは Google Drive 等から直接取得しない
- `rclone` 等でローカルディレクトリへ同期した画像を参照する
- 画像切り替え間隔と再スキャン間隔は設定で変更できる

運用例は [run-on-raspberrypi.md](run-on-raspberrypi.md) を参照してください。

## 設定（JSON + 起動オプション）

設定は JSON ファイルと起動時のコマンドラインフラグを併用できます。

- 既定の設定ファイルパスは `/etc/dashboard-config.json`
- `-config` で別パスの JSON を指定可能
- 優先順位は「既定値 < JSON < フラグ」です

主な設定項目は以下です。

- 天気情報の緯度経度（JSON: `latitude`, `longitude` / フラグ: `-lat`, `-lon`）
- タイムゾーン（JSON: `timezone` / フラグ: `-tz`）
- 画像ディレクトリ（JSON: `photos_dir` / フラグ: `-photos_dir`）
- キャッシュディレクトリ（JSON: `cache_dir` / フラグ: `-cache_dir`）
- 画像の切り替え間隔（JSON: `photo_interval_seconds` / フラグ: `-photo_interval`）
- 画像ディレクトリの再スキャン間隔（JSON: `photo_rescan_interval_seconds` / フラグ: `-photo_rescan_interval`）
- 天気情報の更新間隔（JSON: `weather_interval_minutes` / フラグ: `-weather_interval`）
- 開発用プレビュー出力（JSON: `preview_dir`, `preview_every_ms`, `screen_w`, `screen_h` / フラグ: `-preview_dir`, `-preview_every`, `-screen_w`, `-screen_h`）

JSON は未知の項目を許容しません。設定ミスを早く検知するため、未定義キーがあると起動時にエラーになります。

### 設定ファイル例

```json
{
  "latitude": 35.681236,
  "longitude": 139.767125,
  "timezone": "Asia/Tokyo",
  "photos_dir": "/var/lib/dashboard/photos",
  "cache_dir": "/var/lib/dashboard/cache",
  "photo_interval_seconds": 60,
  "photo_rescan_interval_seconds": 300,
  "weather_interval_minutes": 10
}
```

### 起動例

```bash
./bin/dashboard-linux-armv6 \
  -fb /dev/fb0 \
  -tz Asia/Tokyo
```

上記の例では、ディレクトリや緯度経度などの定常設定は JSON から読み込み、`-tz` だけをフラグで上書きしています。
