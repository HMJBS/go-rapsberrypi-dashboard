# 実装プラン（ドラフト）

このドキュメントは、現状の実装状態を前提に、今後の実装作業の順序と作業範囲を明確にするためのプランです。

- 前提: 現状の仕様（ドキュメント）は [features.md](features.md) に合わせて更新済み
- 方針: Raspberry Pi 1B + FrameBuffer + 低依存を維持（詳細は [architecture.md](architecture.md)）

## 現状（実装済み）

以下はコードで確認できる実装済み機能です。

- 描画ループ（1秒周期でフレーム生成→FB転送 / 代替としてPNGプレビュー出力）
- 時計（7セグで `HH:MM:SS`、毎秒更新）
- 天気取得（Open-Meteo）+ キャッシュ（失敗時は最後のキャッシュ表示）
- 画像表示（ローカルディレクトリをスキャンし、ランダム切替・定期再スキャン）
- 起動設定（フラグで `lat/lon/tz`、各種 interval、`photos_dir`、`cache_dir` 等を指定）
- 設定JSON対応（`/etc/dashboard-config.json` 既定、JSON をベースにフラグで上書き可能）

参考:
- エントリポイント: `cmd/dashboard/main.go`
- 統合ループ/描画: `internal/app/app.go`
- FB: `internal/fb/fb.go`
- 天気: `internal/weather/weather.go`
- 写真: `internal/photos/photos.go`

### P1: 日付表示（YYYY/MM/DD）[完了]

目的:
- 時計だけでなく日付も表示できるようにする（仕様を明確にしやすい）

実装方針:
- `internal/app/app.go` の `render` に日付文字列を追加描画
- 文字描画はフォントを導入し、デザイン案に寄せる
- タイムゾーンは現状どおり `-tz`（IANA名）を使用

注意:
- 現状は `time.Now()` をそのまま使用しているため、タイムゾーン適用が不十分な可能性がある
  - 必要なら `time.LoadLocation` + `now.In(loc)` を導入

### P2: 天気表示の拡張（最高/最低・都市名・アイコン）

目的:
- 画面の情報価値を上げる（現状は「気温＋ラベル」のみ）

推奨アプローチ（低依存・キー不要）:
- Open-Meteoのまま、APIレスポンスを拡張して `daily` の最高/最低を取得
- 都市名はネットワーク依存を増やさず、設定（JSON/フラグ）で表示名を指定する案を優先
- アイコンは「ピクト画像」ではなく、軽量なウィジェット描画（記号/簡易アイコン）で実装

作業メモ:
- `internal/weather/weather.go`

  - キャッシュ形式の後方互換（既存 `weather.json` の読み込み耐性）
- `internal/app/app.go`
  - 表示レイアウト（テキスト行の増加、アイコン領域）
- `internal/widgets`
  - 天気アイコン描画の追加（最小のAPIで）

## リスクと判断ポイント

- 設定をフラグ→JSONへ寄せると運用は楽になる一方、互換性（既存フラグ）とバリデーションが重要
- タイムゾーンを正しく反映するには `time.Location` の扱いを明確にする必要がある
